    In current implementation; first line works for “text/plain” message type. Because approach used is
    1) Take header length as “header_length=_mimeMsg->GetBodyPart()>GetMimeData()>GetHeaderLength()”
    2) And then copy header_length’s next 90 characters to firstline buffer.

    So it does not work for multipart type messages as header_length’s next 90 characters would be “–mime-boundary”.

    For fixing, we would first need to check MainType (pData->GetMainType()) of message,
    1) If its multipart. Then we would iterate over subparts, If text found, take 90 characters from subpart’s BodyOffset.
    2) Else take first 90 characters from BodyOffset.
